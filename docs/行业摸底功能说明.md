# 行业摸底功能说明

## 概述

行业摸底是信息汇总桌面工具的核心功能。输入一个行业名称，系统自动完成搜索→提取→筛选→分析→输出的完整流程，生成一份有角色视角、有在地化分析、有来源分级的行业研究报告，保存到 Obsidian Vault。

## 工作流程

```
输入行业名 → 搜索（中文10路 + 英文静默补充3路）→ 提取全文 → 信源质量评分 →
章节匹配裁剪 → 阶段1：并行生成所有内容章节 →
阶段2：基于完整正文生成摘要 + 来源分级 → 保存 Obsidian → 可选章节重写
```

### 各步骤说明

| 步骤 | 说明 |
|------|------|
| 1. 搜索 | 行业名扩展为10个维度中文查询，并行追加3路英文静默补充（市场趋势/统计数据/创新趋势），每个查询取5条，去重合并 |
| 2. 提取 | 爬取每篇文章全文（HTML→纯文本），成功率约60-80% |
| 3. 评分筛选 | 域名权威度×50% + 内容长度×25% + 时效性×25%，低于0.3剔除，保留前20篇（跨境模式30篇） |
| 4. 英文占比控制 | 静默补充模式下，英文文章不超过总池子25%，超限按质量分截断低分英文 |
| 5. 章节匹配 | 每个章节只喂最相关的6篇文章（关键词+搜索来源匹配），避免token浪费 |
| 6. AI分析 | 两阶段并行生成：先出所有内容章节，再基于完整正文提炼摘要和来源分级 |
| 7. 保存 | 报告写入Obsidian，源文章按扫描时间戳独立建文件夹保存，建立双向链接，更新目录索引 |

## 搜索系统

### 搜索引擎

| 引擎 | 特点 | 需要 |
|------|------|------|
| Tavily | 原生全球搜索，英文结果最佳 | API Key + VPN |
| Bing API | 支持中英双语，自动按查询语言切换市场（中文→zh-CN，英文→en-US） | Azure Key（免费1000次/月），无需VPN |
| DirectEngine | 无需任何Key | 中文→cn.bing.com+百度回退，英文→www.bing.com国际版 |

无Tavily时，Bing API或DirectEngine仍可正常获取海外信息。引擎不可用时自动回退到下一级。

### 查询维度（中文10路）

价值链分析、竞争格局、主要玩家、消费者行为、经营打法、品牌格局、渠道玩法、爆品趋势、竞品拆解、素材定价

### 静默英文补充（普通模式自动开启）

普通模式在中文10路之外，静默并行3条英文查询：
- 全球市场规模与趋势
- 统计数据、增长率、份额
- 创新趋势与新产品

英文结果自动标注 `[英文补充]`，占最终文章池不超过25%。不做章节替换，不注入跨境系统提示——纯粹的数据源补充。

## 出海/跨境模式

搜索栏下方的"出海/跨境"开关，开启后进入跨境模式，影响所有层面：

### 搜索层面

- 中文10路 + 英文10路全量双语搜索
- 英文查询标注 `[英文]`

### 系统提示层面

注入跨境视角引导：目标市场准入条件与关税政策、跨境物流与海外仓、中国供应链优势 vs 海外竞品、两面呈现原则。

### 章节替换

| 原章节 | 跨境版 |
|--------|--------|
| 价值链分析 | **跨境价值链** — 制造→物流→清关→海外仓→终端链路，含关税拆解、"中国+N"产能布局成本对比 |
| 渠道玩法 | **跨境平台对比** — Amazon/Temu/TikTok Shop/独立站/线下分销五平台横向对比，含费用率、GMV规模、政策变化 |

### 章节视角叠加

开启跨境后，以下章节在原提示词上叠加跨境指令（不替换整个章节）：
- **竞争格局** — 追加"中国 vs 海外目标市场"竞争态势对比，中国企业在海外市场实际份额
- **消费者行为** — 追加海内外消费者差异对比，分市场标注对中国品牌接受度、退货习惯、信任因素

## 角色视角系统

同一份搜索素材，6种角色看到完全不同的报告。角色影响三个层面：

| 层面 | 影响 |
|------|------|
| 系统提示词（全局） | 注入角色核心关注问题，重新定义分析框架 |
| 章节提示词 | 追加角色专属分析指令 |
| 章节替换 | 政府视角下"竞争格局"→"产业分布格局"，"经营打法"→"招商策略"（完整替换提示词） |
| 章节跳过 | 政府视角跳过渠道玩法、爆品趋势、竞品打法、素材与定价四个电商章节 |

### 各角色视角

| 角色 | 核心关注 | 适用人群 |
|------|---------|---------|
| **厂家/工厂** | 原材料成本走势、产能规划、技术升级路径、B端大客户拓展 | 生产制造企业 |
| **品牌方** | 品牌定位矩阵、消费者洞察、营销ROI、渠道结构 | 品牌操盘手、市场部 |
| **经销商** | 品牌代理机会、品类空白、区域竞争格局、库存周转 | 代理商、渠道商 |
| **投资人** | 市场规模增速、估值对比、财务数据、退出路径 | VC/PE、产业投资 |
| **政府** | 产业分布格局、招商策略、集群打造、政策建议 | 地方招商部门、园区 |
| **不限** | 通用分析，覆盖全维度 | 其他/探索 |

## 所在地（在地化）

搜索栏下方的"所在地"输入框，填写后所有角色都带上本地视角：

| 角色 | 所在地权重 | 在地化影响 |
|------|-----------|-----------|
| 政府 | 最高 | 招商建议100%结合本地资源禀赋，给出具体对接企业和谈判条件 |
| 厂家 | 高 | 当地原材料供应、物流成本、招工难度、要不要自建配套 |
| 经销商 | 高 | 当地品牌代理饱和度、品类空白、代理空间判断 |
| 投资人 | 中 | 当地产业基金、税收优惠、人才政策对投资回报的影响 |
| 品牌方 | 低 | 全国视角为主，所在地仅作为样板市场或渠道试点参考 |

不填时行为不变；填写后从SYSTEM_PROMPT到每个章节都会注入在地化指引。

## 报告结构

### 行业分析（默认）

| 章节 | 内容 |
|------|------|
| 核心摘要 | 基于完整正文提炼，包含核心发现、关键数字、三大趋势、可考虑方向（附前置条件）、最大风险 |
| 价值链分析 | 上中下游全景、利润池迁移、各环节盈利能力对比 |
| 竞争格局 | 市场集中度、头部策略、新进入者壁垒、替代品威胁、最大风险标注 |
| 主要玩家 | Top企业对比表（优势/短板/战略/动态），含数据来源标注 |
| 消费者行为 | 用户画像、决策路径、客单价、已知+隐性痛点矩阵 |
| 经营打法 | 商业模式、获客方式、供应链效率、可迁移实操技巧 |
| 品牌格局 | 头部+新锐品牌矩阵、差异化定位、市场空位分析 |
| 渠道玩法 | 线上线下渠道结构、抖音/小红书/直播/投放策略 |
| 爆品与趋势 | 热销品类、趋势方向、卖点拆解、销量数据 |
| 竞品打法拆解 | 竞品内容拆解、话术分析、投放关键词 |
| 素材与定价 | 视觉方向、文案策略、定价带、促销机制 |
| 来源分级 | 核心必读/深度参考/延伸阅读三级，含来源类型标注（[英文]/[英文补充]） |

### 产品/爆款分析（系统自动识别）

| 章节 | 内容 |
|------|------|
| 核心摘要 | 同上，偏产品归因 |
| 爆款归因 | 走红原因拆解、传播路径、关键引爆节点 |
| 人群画像 | 购买人群特征、使用场景、复购驱动 |
| 竞品格局 | 同类产品对比、替代品、差异化定位 |
| 热度周期 | 生命周期判断、增长阶段、衰退预警 |
| 电商切入 | 选品策略、供应链对接、开店成本测算 |
| 内容营销 | 种草打法、内容模版、KOL矩阵、投放预算参考 |

## 质量标准：两面呈现

所有报告遵循"两面呈现"原则，核心要求：

- **禁止绝对化表述**：用"数据显示X在Y条件下更优"替代"X最好"，用"在Z情况下可考虑W"替代"必须做W"。禁止"最值得""必将""唯一的出路""不二之选"等词
- **利弊并陈**：任何方向性判断必须同时列出支持数据和反对条件
- **条件化建议**：核心摘要的"可考虑方向"每条附带前置条件说明，不做无条件的行动推荐
- **风险标注**：涉及高风险判断的章节末尾标注"本分析基于当前可得信息，若市场环境或政策条件发生重大变化，部分结论可能需要重新评估"

核心理念：**本工具是信息整理者，不是决策替代者**。整理信息帮助读者做判断，不替读者做决定。

## 信源质量

搜索到的每篇文章经过质量评分（0-1分）：

- **域名权威度**（50%）：gov.cn、券商、顶级媒体→0.8-1.0；行业媒体→0.6-0.7；自媒体/论坛→0.3-0.4
- **内容长度**（25%）：<200字→0.1，200-500→0.4，500-1000→0.7，>1000→1.0
- **时效性**（25%）：未识别年份→0.5，2026→1.0，2025→0.9，2024→0.7，2023→0.5，更早→0.3

筛选规则：
- 低于0.3分剔除
- 保留前20篇（跨境模式30篇）
- 分数<0.5的文章内容截断至1000字符，分数<0.7截断至1500字符
- 静默英文补充占比不超过25%

## 文件保存

### 报告

保存在 `{Obsidian Vault}/行业摸底/{行业名}/`，文件名格式 `{行业名} 行业分析报告 (2026-05-31 165354).md`。

### 源文章

保存在 `{Obsidian Vault}/行业摸底/{行业名}/sources/{扫描时间戳}/`。每次扫描创建独立子文件夹，不同批次的原文互不混杂。

### 索引

`{Obsidian Vault}/行业摸底/行业摸底索引.md` 自动更新，记录所有历史报告。

## 章节重写

报告完成后，在成功提示卡片下方的下拉菜单中选择章节即可重写。重写使用原始缓存的搜索结果（30分钟有效），按章节重新匹配上下文后生成。

适用场景：某章节不满意，换个角度重写，无需重新跑全流程。

## Token用量参考

以"建材行业"为例（政府视角，8个内容章节 + 摘要 + 来源分级）：

- 总Token：约12-13万
- 提示Token：约10.5-11万（含章节匹配上下文 + 提示词）
- 生成Token：约1.7-1.8万
- 估算费用：约¥0.14-0.15（DeepSeek）

章节匹配裁剪后，每章上下文从全量降至6篇，相比裁剪前节省约40-50%提示Token。

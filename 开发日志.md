### 1. 知识库搜索找不到内容

**现象：** 用户在策略对谈问"知识库有海尔相关的内容吗"，明明已索引 608 篇文章，返回结果中 0/15 条与海尔相关。

**根因：** `paraphrase-multilingual-MiniLM-L12-v2` 嵌入模型对中文短查询理解弱。用户用自然语言提问，包含大量噪声词（"知识库""有没有""能不能"），语义搜索被噪声带偏。

**解决方案：** 在策略对谈的 `/api/strategy/chat` 中增加 LLM 关键词提取步骤 `_extract_search_keywords()`。先用轻量 prompt 把"知识库有海尔相关的内容吗"提取为"海尔 家电 经销商 电商"，再用提取后的关键词做语义搜索。

**涉及文件：** `backend/routes/strategy.py`

### 3. 启动时自动同步知识库 + 预加载

**需求：** 用户希望每次启动应用自动同步 Obsidian vault 到向量数据库。

**实现：**
- 后端 `PUT /api/config` 支持 `sync_on_startup` 字段（`backend/routes/config.py`）
- 前端设置页增加"启动时自动同步知识库"开关（`src/pages/Settings.tsx`）
- `backend/main.py` lifespan 中：先同步等待模型加载完毕，再后台异步执行同步

**额外优化：** 将 preload 从 `run_in_executor`（不等待）改为 `await run_in_executor`（等待完成），避免启动时 model 未就绪就接收搜索请求导致报错。

### 4. 知识库分层：报告 vs 原始文章

**需求：** 用户提出——报告和卡片的标题很具体（如"海尔家电行业分析"），搜索匹配度天然高。608 篇文章里大量是原始抓取的网页文章，它们混在向量库里反而稀释了搜索精度。

**解决方案：** `sync_vault_to_collection()` 增加 `exclude_sources` 参数，默认跳过 `sources/` 目录下的原始文章，只索引报告和知识卡片。索引量从 608 → 44 篇，搜索命中率大幅提高。

**涉及文件：** `backend/services/embedding_service.py`, `backend/routes/embeddings.py`, `src/pages/Settings.tsx`

### 5. 未知领域 → 一键跳转行业摸底

**场景：** 用户在策略对谈问到知识库中没有的领域（如"建材行业"），搜索结果为空或无关。之前只能手动复制关键词切到行业摸底页面。

**解决方案：**
- 后端在知识搜索无结果或结果不相关时，通过 SSE 发送 `suggest_scan` 事件，附带 `topic` 字段
- 前端收到事件后渲染"去行业摸底生成「XXX」分析报告"按钮
- 点击按钮跳转到行业摸底页面并自动填入关键词

**关键细节：** 用 `_generic_words` 过滤泛词（"行业""报告""市场"等），避免标题中出现了这些词就被误判为"相关"。

**涉及文件：** `backend/routes/strategy.py`, `src/pages/StrategyDialog.tsx`, `src/App.tsx`

### 6. 行业摸底预填关键词不生效

**现象：** 从策略对谈跳转过来后，搜索栏是空的，关键词没填进去。

**根因：** React 的 `useState` 初始值只在组件首次挂载时生效。策略对谈和行业摸底用 `display: none/block` 切换，组件不会重新挂载。

**修复：** 在 `IndustryScan.tsx` 中加 `useEffect` 监听 `prefillKeyword` prop 变化，变化时同步更新 `industry` state。

**涉及文件：** `src/pages/IndustryScan.tsx`

### 7. ChromaDB 损坏自动恢复

**问题：** 开发过程中多次 `kill -9` Python 进程导致 ChromaDB SQLite 文件损坏，出现"Cannot open header file"、"Could not connect to tenant"等错误。

**解决方案：** 在所有 ChromaDB 操作处（`get_chroma_client`, `get_collection`, `search`, `index_articles`, `sync_vault_to_collection`）包装 try/except + 自动恢复逻辑：
1. 捕获已知的损坏错误模式（8 种）
2. 自动删除损坏的 `chroma_db_v3/` 目录
3. 重置全局状态后重试

**涉及文件：** `backend/services/embedding_service.py`

**遗留任务：** `data/chroma_db/` 旧目录被僵尸进程锁定，需重启电脑后手动删除。

### 8. 行业 vs 产品自动分类

**需求：** 用户可能查询行业（"家电""宠物经济"），也可能查询具体产品（"拉布布""筋膜枪"）。两者分析框架完全不同——行业需要产业链、竞争格局，产品需要爆款归因、人群画像。

**实现：**
- `_classify_topic()` 做一次轻量 LLM 调用判断输入类型
- 产品走 6 章节框架（爆款归因→人群画像→竞品格局→热度周期→电商切入→内容营销）
- 行业走 10 章节框架（价值链→竞争格局→…→素材与定价）
- 进度条显示"识别为: 产品/爆款分析"

**设计决策：** 用 LLM 而非关键词规则分类，因为用户输入形式不可预测（"最近很火的那个盲盒娃娃""想做这个赛道"等）。

**涉及文件：** `backend/services/ai_analyzer.py`, `backend/models/prompts.py`

### 9. Windows 打包（499MB NSIS 安装程序）

**目标：** 让朋友无需安装 Python 环境就能使用。

**方案：** PyInstaller 打包后端 + electron-builder 打包前端 + Electron 运行时 → NSIS 安装程序。

**涉及文件：** `pyinstaller.spec`, `electron/main.ts`, `package.json`, `scripts/build.bat`

**已知问题：** Windows 上 uvicorn `--reload` 的 watchfiles 不检测文件变化，开发时需手动重启后端。

### 10. Windows 开发环境热重载不生效

**现象：** 修改代码后 uvicorn 不自动重启。

**根因：** watchfiles 1.2.0 在 Windows 上有兼容问题，文件变化事件不触发。

**临时方案：** 每次改代码后手动 `kill` Python 进程再重启。
